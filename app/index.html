[ recipes on a shelf ]

A shelf for the scripts your agents keep re-deriving.

pantry is a small Cloudflare Worker and D1 store of recipes, so an agent can reuse a saved recipe instead of re-reasoning the same pattern. It stores the code and hands it back when asked. It never runs it.

The recipe shape

A recipe is a named JavaScript function body with an input schema and a list of capability tags. pantry keeps it in D1 and returns the code verbatim when asked.

Cheap discovery

GET /recipes lists names, descriptions, schemas, and capabilities without the code, so a model can decide a recipe fits before transferring the script.

It never runs your code

pantry stores and transports. Running a fetched recipe, and in what isolate, is the caller's trust decision. The example runner is a convenience, not a sandbox.

How a recipe gets used

discover    GET /recipes        names + schemas, no code
fetch       GET /recipe/:name   the full recipe, including code
decide      caller's choice     run it, or re-reason it
run         caller's isolate    pantry is not involved

A saved recipe saves model tokens only for a recurring pattern. Novel work still needs reasoning, because there is no saved recipe to reuse.
