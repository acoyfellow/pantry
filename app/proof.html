Proof, with labels.

pantry's honest claim is structural: a model can re-derive a recurring procedure every time and may be wrong; pantry hands back exact saved code and spends about 0 model tokens regenerating that saved code.

Headline from this repo's eval

The local harness covers four recurring tasks. In the current checked-in run, no live model provider was reachable, so prompt-from-scratch and inline-tool numbers are tokenizer-measured payload sizes rather than provider spend. Pantry recipe execution used 0 tokens regenerating saved code, measured local deterministic execution, and passed the deterministic oracle for 4 of 4 tasks.

The structural claim now has a live-provider backing measurement from the my-ax saved-recipe loop: real Workers AI Kimi K2.7 on 2026-06-29 cut output tokens about 72% when the same procedure was reused as saved code instead of regenerated. A 2026-06-30 fresh-session attribution run measured 394 output tokens deriving from scratch and 157 via recipe.run, while total tokens rose for that tiny procedure because invoking the recipe added input/tool overhead.

What is measured

The harness in evals/run.ts uses real recipes: slugify, verify_live_worker, deploy_coey_worker, and one deterministic JSON transform. Pantry reuse runs saved recipe code locally, records wall-clock milliseconds, and checks the output with the same oracle used for live model answers.

Default mode does not pretend to be a live model run. It uses gpt-tokenizer for prompt and discovery payload sizes. Live mode is gated behind LIVE_MODEL=1 and only reports provider usage when a provider actually returns usage.

Limits

  • This applies to recurring patterns with a saved recipe. Novel work still needs reasoning.
  • There is always discovery cost. The agent must inspect recipe names, descriptions, schemas, and capabilities.
  • Tokenizer counts are payload sizes, not provider billing, unless live mode records provider usage.
  • Local deterministic wall-clock is not network latency and not a live model timing result.
  • pantry stores and hands back code. Running fetched code remains the caller's trust decision.
